Como eu Lido com Updates do Arch Linux Repo Fixo Snapshot


Description

Veja meu arquivo de mirrorlist no github: https://github.com/mountaineerbr/dotfiles/blob/master/etc/pacman.d/mirrorlist

Dicas
-- Faça updates somente quando precisar dos pacotes mais novos/recursos mais novos; -- Só instale novos pacotes depois de fazer um upgrade total no sistema, ou seja, antes de instalar um novo pacote, sempre atualize o sistema antes; -- Há períodos no mês em que os updates podem ser mais estáveis que outros; minha observação indica que os pacotes estão mais estáveis imediatamente antes da criação mensal de um novo .iso de instalação do Arch Linux pelos Desenvolvedores. A c -- Depois de fazer uma atualização no sistema e deixar ele estável, você poderá usar um repositório "fixo" (snapshot) por um tempo para não ter que atualizar o sistema todo caso precise instalar um pacote novo. Nesse caso, vá até o seu arquivo de

Server=https://archive.archlinux.org/repos/2019/08/31/$repo/os/$arch

Essa linha indica para o Pacman usar o repositório fixo do dia que você especificar. No exemplo acima, o repo fixo é do dia 31 de agosto de 2019.

OBS: se você for sincronizar com um repo mais antigo do que a data atual, será necessário usar -Syy para forçar o update da lista de pacotes do Pacman, além de precisar fazer um downgrade em todos os pacotes do sistema!

Referência:
https://www.ostechnix.com/downgrade-packages-specific-date-arch-linux/ https://archive.archlinux.org/repos/2019/08/31/

O arquivo de Mirrorlist/Espelhos encontra-se em: /etc/pacman.d/mirrorlist

IMPORTANTE: Os pacotes do AUR não são suportados oficialmente e você estará sob seu próprio risco ao instalar eles!
