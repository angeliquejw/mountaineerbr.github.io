Arch Updates Mais Dicas


Description

Dicas
1. Conheça o ALA (Arch Linux Archive), que é um repositório da comunidade oficial que todos podem usar para fazer downloads de pacotes de versões anteriores, Ref(I).

Em caso de precisar fazer um downgrade de um pacote ou de todos os pacotes do sistema, você vai precisar do ALA se não tiver os binários antigos no seu disco..

2.Faça o download completo dos repositórios de um dia específico (~40GB). Por exemplo, você pode fazer o download de todos os arquivos dos repos core, extra, community e multilib, etc, de um dia específico.

OBS: Mantenha a Estrutura do Repositório!

Uma vez no seu disco, verifique se também foi feito o download do arquivo de base de dados (.db). Se você for fazer um repositório totalmente customizado, ou seja, adicionar pacotes além dos que você fez o download do ALA(snapshot), vai precisar rec

[XXNOME_REPOXX]
SigLevel = Never
Server = file:///caminho/repo/quefez/download

em cada bloco de opções para o core, extra, community e multilib(se usar) no arquivo /etc/pacman.conf.

Há várias maneiras de se configurar esse uso dos repos que você fez o download com o pacman.

3.Configure seu pacman.conf para sincronizar com os repos mensais ou semanais do Arch Linux.

Para isso, vá até o arquivo /etc/pacman.d/mirrorlist e adicione a opção a seguir no topo da lista.

Para updates mensais:

Server=https://archive.archlinux.org/repos/month/$repo/os/$arch

Para updates semanais:

Server=https://archive.archlinux.org/repos/week/$repo/os/$arch
